编程风格

  • 声明式编程和命令式编程站的角度不同，一个关注的式结果，一个关注的是过程，声明式编程对于接口的使用者来说是非常好的，但是接口的内部还是得要用命令式编程来实现。
  • 命令式 Imperative 编程
    • 关注：计算机执行的具体步骤。即先做什么， 再做什么
    • 比如在 列表中筛选大于 2 的数字，需要告诉它：先创建一个结果列表、再遍历这个列表，再判断每个数字是否大于 2，如果是就添加到结果变量列表中。
  • 声明式 Declarative 编程
    • 关注：最后的结果，以数据结构的形式来表达程序执行的逻辑。
    • 告诉计算机应该做什么，但不指定具体要怎么做
    • HTML、CSS 也是声明式的
    • 它不需要创建变量用来存储数据。 另一个特点是它不包含循环控制的代码如 for，while。
  • 函数式编程
    • 关注：
    • 在声明式编程上更进一步，将函数作为第一公民
    • 函数式编程和声明式编程是有所关联的，因为他们思想是一致的：即只关注做什么而不是怎么做。但函数式编程不仅仅局限于声明式编程。 函数式编程最重要的特点是**“函数第一位”**，即函数可以出现在任何地方，比如你可以把函数作为参数传递给另一个函数，不仅如此你还可以将函数作为返回值。大部分常见的编程语言一半都已经提供了对这种编程方式的支持，比如 JavaScript，再有 C# 中的 LINQ 和 Java 中的 Lambda 和闭包的概念。Java 8 最大的一个对函数式编程支持的更新就是 Stream API，

C

C：主要用于开发底层模块 (如驱动、解码器、算法实现)、服务应用（如 web 服务器）、嵌入式开发。

  • 如果不了解 C 或 C ++，你就不能成为程序员。这是一个非常强烈的声明
  • 知道 C 的软件工程师或开发人员比不懂 C 的程序员更好，而且不仅仅是巧合。

C++

C++：游戏开发, 大规模, 高性能, 分布式要求的程序开发。

  • 适用于复杂但又需要高效率的开发。

Java：网页，企业级开发，过去二十年来最流行的语言之一，具有好的语言特性，丰富的框架。

Android 开发、企业级 web 开发, 普通应用软件, 游戏后台。
