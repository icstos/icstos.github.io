服务相关

进入服务：win+r–services.msc

  • 关闭Windows Search：WindowsSearch 右键 - 启动类型–禁用。
  • 可选择性关闭
    • 关闭程序兼容助手服务：Program CompatibilityAssistant Service服务 禁用。
    • 【已没有】关闭家庭组：**HomeGroup Listener 和 HomeGroup Provider**禁用。
    • 禁Superfetch服务（现在改名叫 SysMain 了）
      • SuperFetch 超级预读服务主要为企业应用与大型协作软件设计，个人用户没必要开启。主要用来对那种 超大软件的开启需长达 10 分钟的企业软件，降低到几分钟风。而开启服务的话，只会加大对硬盘的监测（损失 CPU 性能时间）与读取时间（增加硬盘 IO 寻址线程分配），这也是大家抱怨硬盘占用大的根本原因。
      • 对于机械硬盘有一定作用，但固态硬盘作用不大，而且常常是导致 CPU 占用飙升 100% 的元凶。如果用户使用的是 SSD，且物理内存较大，可以考虑关闭。
    • “Connected User Experiences and Telemetry”、“Diagnostic Execution Service”、“Diagnostic Service Host”、“Diagnostic System Host” 禁用
    • 付款和 NFC/SE 管理
    • `web 账户管理器
      在搜索栏中输入“service”来优化系统服务。其中“Connected User Experiences and Telemetry”、“Diagnostic Execution Service”、“Diagnostic Policy Service”、“Diagnostic Service Host”、“Diagnostic System Host”是 Win10 的诊断服务，用处不大，关闭后能明显提高系统响应速度，其他服务则可以根据自己的需要调整。

关闭 DIPM 节能机制

DIPM：Serial ATA DIPM，标准解释是串口额外省电模式。

  • 在系统空闲时，通过减少硬盘串口供电，来让 SSD 进入低功耗节能状态。但不太稳定，导致电脑出现卡顿，过 1-2 秒后又正常，电，不如使用流畅。
  • BIOS 关闭：Serial ATA DIPM
  • 操作系统“电源选项”中的“PCI Express”→“链接状态电源管理”也会触发 SSD 的节能模式

环境变量

  • 优先级：
    • 同名变量，用户变量 会覆盖 系统变量，but，对于 Path 这个变量，是会将 用户变量的 Path 添加到系统变量的后面
    • windows 在查找某一指令的时候，是按 Path 中的路径从前往后找，直至在某个路径中找到了该指令，所以说，如果系统变量 Path 中有对应指令的话，windows 就不会再去用户变量 Path 中查找
      • 简单说：对于 Path 变量，系统变量的优先级高于用户变量，会先在系统变里找 ，找不到才到用户变量里找
  • Windows 的环境变量优先级，PATH 中的是上面的优先级高（比如 Python 的环境，先定义的为准）

磁盘管理命令可让您灵活地直接从命令行管理存储资源，为图形用户界面工具提供强大的替代方案。

echo: 打开回显或关闭请求回显功能，或显示消息。如果没有任何参数，echo 命令将显示当前回显设置。

  • rem: 注释命令，类似于在 C 语言中的/——–/，它并不会被执行，只是起一个注释的作用，便于别人阅读和你自己日后修改。

  • set: 显示、设置或删除变量。

基本命令

echo 与@

  • &：顺序执行多条命令，而不管命令是否执行成功

  • &&：顺序执行多条命令，当碰到执行出错的命令后将不执行后面的命令

    • find “ok” c:test.txt && echo 成功

  • ||：顺序执行多条命令，当碰到执行正确的命令后将不执行后面的命令

    • find “ok” c:test.txt || echo 不成功

  • |：管道命令：dir . /s/a | find /c “.exe”：管道命令表示先执行 dir 命令，对其输出的结果执行后面的 find 命令

  • >：清除文件中原有的内容后再写入

  • >>：追加内容到文件末尾，而不会清除原有的内容

  • <：从文件中获得输入信息，而不是从屏幕上，一般用于 date time label 等需要等待输入的命令

  • if：

  • set：设置变量，引用变量可在变量名前后加 % ，即 % 变量名%

  • start：批处理中调用外部程序的命令，否则等外部程序完成后才继续执行剩下的指令

  • call：批处理中调用另外一个批处理的命令，否则剩下的批处理指令将不会被执行，有时有的应用程序用 start 调用出错的，也可以 call 调用

  • assoc 和 ftype：

    • assoc 设置 ‘ 文件扩展名 ‘ 关联，关联到 ‘ 文件类型 ‘
    • ftype 设置 ‘ 文件类型 ‘ 关联，关联到 ‘ 执行程序和参数 ‘

  • for：循环命令，for %%i in (c: d: e: f:) do echo %%i
